# Prezentare generală API

CuibOS expune un API REST prin intermediul unui **Cloudflare Edge Gateway** care rutează cererile către microserviciile interne.

## URL de bază

```
https://api.cuibos.ro
```

## Autentificare

Toate endpoint-urile protejate necesită un **JWT Bearer token** în header-ul `Authorization`:

```http
Authorization: Bearer <access_token>
```

Token-ul se obține prin endpoint-ul `/auth/v1/login` și are durata standard de 15 minute. Reînnoirea se face prin `/auth/v1/refresh` cu un refresh token (cookie httpOnly).

## Header-e standard

| Header | Obligatoriu | Descriere |
|---|---|---|
| `Authorization` | Da (protejat) | `Bearer <JWT>` |
| `Content-Type` | Da (POST/PUT/PATCH) | `application/json` |
| `X-Request-Id` | Nu | ID unic pentru corelarea cererilor (auto-generat dacă lipsește) |
| `X-Locale` | Nu | Limba preferată: `ro` sau `en` |
| `Accept-Language` | Nu | Alternativă pentru locale |

## Header-e injectate de gateway

Gateway-ul Cloudflare extrage claim-urile din JWT și le atașează ca header-e downstream:

| Header | Sursa | Descriere |
|---|---|---|
| `X-User-Id` | `payload.sub` | ID-ul utilizatorului |
| `X-User-Email` | `payload.email` | Email-ul utilizatorului |
| `X-Tenant-Id` | `payload.tenantId` | ID-ul tenant-ului |
| `X-Role` | `payload.role` | Rolul: owner, admin, viewer |
| `X-Plan` | `payload.plan` | Planul: free, pro, pro_plus |

## Rutarea serviciilor

| Prefix gateway | Serviciu intern |
|---|---|
| `/auth/v1/` | auth-service |
| `/properties/` | property-service |
| `/vehicles/v1/` | vehicles-service |
| `/finance/v1/` | finance-service |
| `/billing/v1/` | billing-service |
| `/notify/v1/` | notify-service |

## Endpoint-uri publice

Următoarele endpoint-uri **nu necesită JWT**:

- `POST /auth/v1/login`
- `POST /auth/v1/register`
- `POST /auth/v1/register-paid`
- `POST /auth/v1/verify-email`
- `POST /auth/v1/forgot-password`
- `POST /auth/v1/reset-password`
- `POST /auth/v1/refresh`
- `POST /auth/v1/introspect`
- `POST /auth/v1/accept-invite`
- `GET /auth/v1/jwks.json`
- `POST /billing/v1/public/checkout`
- `GET /billing/v1/public/checkout-session/:sessionId`
- `POST /billing/v1/webhooks/stripe`

## Coduri de eroare

| Cod | Descriere |
|---|---|
| `401` | Token lipsă sau expirat |
| `403` | Rol insuficient |
| `422` | Eroare de validare (câmpuri invalide) |
| `404` | Resursă negăsită |
| `409` | Conflict (resursă duplicat) |
| `500` | Eroare internă server |

## Convenții răspuns

- **ID-urile** sunt UUID-uri v4
- **Datele** sunt în format ISO 8601 (`2025-01-15T10:30:00Z`) sau `YYYY-MM-DD` pentru câmpuri date-only
- **Sumele** sunt serializate ca string-uri (ex: `"129.50"`) pentru precizie decimală
- **Enum-urile** sunt string-uri lowercase
