# Auth Service

Prefix: `/auth/v1`

Serviciul de autentificare gestionează conturile, sesiunile JWT, verificarea email-ului, invitațiile de echipă și parolele.

## Înregistrare și autentificare

### POST /auth/v1/register

Creează un cont nou (plan gratuit).

::field-group
  ::field{name="email" type="string" required}
  Adresa de email
  ::
  ::field{name="password" type="string" required}
  Parola (minim 8 caractere)
  ::
  ::field{name="name" type="string"}
  Numele afișat
  ::
  ::field{name="locale" type="string"}
  Limba preferată (`ro` / `en`)
  ::
::

---

### POST /auth/v1/register-paid

Creează un cont asociat unui checkout Stripe deja plătit.

::field-group
  ::field{name="email" type="string" required}
  Adresa de email (trebuie să coincidă cu cea din Stripe checkout)
  ::
  ::field{name="password" type="string" required}
  Parola
  ::
  ::field{name="name" type="string"}
  Numele afișat
  ::
  ::field{name="checkoutSessionId" type="string" required}
  ID-ul sesiunii Stripe checkout
  ::
::

---

### POST /auth/v1/login

Autentificare cu email și parolă. Returnează access token și setează refresh token ca cookie httpOnly.

::field-group
  ::field{name="email" type="string" required}
  Adresa de email
  ::
  ::field{name="password" type="string" required}
  Parola
  ::
::

**Răspuns:**
```json
{
  "accessToken": "eyJhbGciOi...",
  "user": {
    "id": "uuid",
    "email": "user@example.com",
    "name": "Cristian",
    "role": "owner",
    "plan": "pro",
    "tenantId": "uuid"
  }
}
```

---

### POST /auth/v1/logout

Invalidează sesiunea curentă și șterge refresh token cookie.

---

### POST /auth/v1/refresh

Reînnoiește access token-ul folosind refresh token din cookie.

**Răspuns:**
```json
{
  "accessToken": "eyJhbGciOi..."
}
```

---

### POST /auth/v1/introspect

Verifică și decodifică un access token. Util pentru validare client-side.

---

### POST /auth/v1/update-profile

🔒 **Necesită JWT**

## JWKS

### GET /auth/v1/jwks.json

Returnează setul de chei publice (JWKS) folosite pentru verificarea JWT-urilor. Folosit intern de gateway.

```json
{
  "keys": [
    {
      "kty": "RSA",
      "kid": "...",
      "n": "...",
      "e": "AQAB",
      "alg": "RS256",
      "use": "sig"
    }
  ]
}
```
