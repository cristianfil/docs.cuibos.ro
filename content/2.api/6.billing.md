# Billing Service

Prefix: `/billing/v1`

Serviciul de facturare gestionează abonamentele prin integrarea cu Stripe.

## Checkout

### POST /billing/v1/checkout

🔒 **Necesită JWT** (owner only)

Creează o sesiune Stripe Checkout pentru upgrade-ul planului.

::field-group
  ::field{name="planName" type="string" required}
  Numele planului: `pro` sau `pro_plus`
  ::
::

**Răspuns:**
```json
{
  "url": "https://checkout.stripe.com/c/pay/..."
}
```

---

### POST /billing/v1/public/checkout

Creează o sesiune Stripe Checkout **fără autentificare** (pentru fluxul de înregistrare cu plan plătit).

::field-group
  ::field{name="planName" type="string" required}
  Numele planului: `pro` sau `pro_plus`
  ::
  ::field{name="email" type="string"}
  Email-ul clientului (opțional)
  ::
::

---

### GET /billing/v1/public/checkout-session/:sessionId

Returnează detaliile unei sesiuni de checkout (public, pentru verificare post-plată).

---

## Portal Stripe

### POST /billing/v1/portal

🔒 **Necesită JWT** (owner only)

Generează un link către portalul Stripe pentru gestionarea abonamentului.

**Răspuns:**
```json
{
  "url": "https://billing.stripe.com/p/session/..."
}
```

---

## Abonament

### GET /billing/v1/subscription

🔒 Returnează detaliile abonamentului curent.

**Răspuns:**
```json
{
  "plan": "pro",
  "status": "active",
  "currentPeriodEnd": "2025-02-15T00:00:00Z",
  "cancelAtPeriodEnd": false
}
```

---

## Facturi abonament

### GET /billing/v1/invoices

🔒 Returnează facturile Stripe ale tenant-ului.

---

## Webhooks

### POST /billing/v1/webhooks/stripe

Endpoint pentru webhook-urile Stripe. **Nu necesită JWT** — autentificat prin semnătura Stripe.

Evenimente gestionate:
- `checkout.session.completed` — activarea abonamentului
- `customer.subscription.updated` — schimbări plan
- `customer.subscription.deleted` — anulare abonament
- `invoice.payment_succeeded` — plată reușită
- `invoice.payment_failed` — plată eșuată
