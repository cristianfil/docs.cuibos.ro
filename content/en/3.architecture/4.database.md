# Databases

CuibOS uses the **database per service** pattern — each microservice has its own isolated PostgreSQL database, managed through **Neon** (serverless Postgres).

## Schema per service

### auth-service

Main tables:
- `users` — user accounts
- `tenants` — organizations (family accounts)
- `tenant_members` — user-tenant associations
- `invitations` — team invitations
- `refresh_tokens` — active sessions
- `verification_tokens` — email tokens
- `password_reset_tokens` — password reset tokens

### property-service

Main tables:
- `properties` — real estate properties
- `property_users` — user access to properties (landlord, tenant, manager, viewer)
- `property_utilities` — utilities (electricity, gas, water, heating)
- `utility_meters` — individual meters
- `meter_readings` — meter readings
- `meter_cycles` — calculated consumption cycles
- `property_services` — recurring services
- `service_catalog_items` — predefined service catalog
- `rental_agreements` — rental contracts
- `event_outbox` — transactional outbox

### vehicles-service

Main tables:
- `vehicles` — vehicles
- `vehicle_expenses` — vehicle expenses
- `odometer_readings` — mileage readings
- `maintenance_items` — maintenance planning
- `maintenance_logs` — maintenance log
- `event_outbox` — transactional outbox

### finance-service

Main tables:
- `budgets` — family budgets
- `budget_lines` — budget lines
- `bills` — bills
- `installments` — installments
- `installment_payments` — installment payments

### billing-service

Main tables:
- `subscriptions` — Stripe subscriptions
- `signup_checkouts` — pre-registration checkout sessions
- `property_usage_states` — property tracking per tenant
- `vehicle_usage_states` — vehicle tracking per tenant
- `usage_sync_jobs` — usage sync with Stripe

## Neon (Serverless PostgreSQL)

Each service connects to a separate Neon branch:
- **Connection pooling** — Neon provides native PgBouncer
- **Autoscaling** — compute scales automatically
- **Branching** — ability to create branches for testing

## Prisma ORM

All services use **Prisma** as ORM:

- `prisma/schema.prisma` — model definitions
- `prisma migrate dev` — development migrations
- `prisma migrate deploy` — production migrations
- `prisma generate` — TypeScript client generation

### Conventions

- **UUIDs** for all primary keys
- **Soft delete** with `deletedAt` (nullable timestamp)
- **Automatic timestamps**: `createdAt`, `updatedAt`
- **Enums** defined in Prisma schema, mapped to PostgreSQL native enums
- **Decimal** stored as Prisma `Decimal` → serialized as string in JSON
