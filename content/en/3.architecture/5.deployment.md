# Deployment

## Infrastructure

CuibOS runs on a **K3s** cluster (lightweight Kubernetes) on **Hetzner Cloud**, with edge routing through **Cloudflare**.

```
┌─────────────────────┐     ┌──────────────────────┐
│  Cloudflare CDN     │     │  Vercel              │
│  Workers + Access   │     │  (Next.js frontend)  │
│  (api.cuibos.ro)    │     │  (app.cuibos.ro)     │
└──────────┬──────────┘     └──────────────────────┘
           │
           │ CF Access Tunnel
           ▼
┌──────────────────────────────────────┐
│  Hetzner Cloud                       │
│  ┌────────────────────────────────┐  │
│  │  K3s Cluster                   │  │
│  │                                │  │
│  │  Namespace: cuibos             │  │
│  │  ┌──────────────────────────┐  │  │
│  │  │ auth-service (Pod)       │  │  │
│  │  │ billing-service (Pod)    │  │  │
│  │  │ property-service (Pod)   │  │  │
│  │  │ vehicles-service (Pod)   │  │  │
│  │  │ finance-service (Pod)    │  │  │

```

## External Services

| Service | Role | Provider |
|---|---|---|
| **Neon** | Serverless PostgreSQL | neon.tech |
| **Stripe** | Payments and subscriptions | stripe.com |
| **Cloudflare Workers** | Edge gateway | cloudflare.com |
| **Cloudflare KV** | JWKS cache | cloudflare.com |
| **Cloudflare Access** | Secure tunnel | cloudflare.com |
| **Vercel** | Frontend hosting | vercel.com |
| **Resend** | Email delivery | resend.com |

## CI/CD

Each microservice has a **GitHub Actions** pipeline:

1. **Push to `main`** → trigger workflow
2. **Build** → Docker image
3. **Push** → GitHub Container Registry (ghcr.io)
4. **Deploy** → `kubectl apply` on K3s cluster

### Gateway

The Cloudflare Worker gateway has a separate workflow:
1. Push to `main` → `wrangler deploy`

### Frontend

The frontend (app.cuibos.ro) deploys automatically on **Vercel** on push to `main`.

## Secrets

| Secret | Where | Description |
|---|---|---|
| `DATABASE_URL` | K8s Secret per service | Neon connection string |
| `RABBITMQ_URL` | K8s Secret | Internal RabbitMQ URL |
| `STRIPE_SECRET_KEY` | K8s Secret (billing) | Stripe secret key |
| `STRIPE_WEBHOOK_SECRET` | K8s Secret (billing) | Webhook secret |
| `JWKS_PRIVATE_KEY` | K8s Secret (auth) | RSA private key |
| `CF_ACCESS_CLIENT_ID` | Worker Secret | Cloudflare Access |
| `CF_ACCESS_CLIENT_SECRET` | Worker Secret | Cloudflare Access |

## JWKS Key Rotation

JWKS key rotation is done through a dedicated script:

1. Generate a new RSA key pair (2048 bits)
2. Update JWKS in Cloudflare KV
3. Update private key in auth-service
4. Old keys remain in JWKS for validating existing tokens
