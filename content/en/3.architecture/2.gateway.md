# Edge Gateway

The CuibOS gateway is a **Cloudflare Worker** running at the edge (closest datacenter to the user).

## Responsibilities

1. **Request routing** — directs URL prefixes to internal services
2. **JWT verification** — validates token with JWKS from Cloudflare KV
3. **Claims injection** — extracts data from JWT and attaches as headers
4. **RBAC** — checks user role for certain routes
5. **CORS** — handles preflights and cross-origin headers
6. **Locale** — propagates preferred language from `X-Locale` or `Accept-Language`

## Request Flow

```
Client
  │
  ├─ OPTIONS (preflight) → 204 with CORS headers
  │
  └─ GET/POST/... with Authorization: Bearer <JWT>
        │
        ├─ Public path? → proxy directly to service
        │
        └─ Protected path:
              ├─ Missing token → 401
              ├─ Invalid/expired token → 401
              ├─ Insufficient role → 403
              └─ Valid → proxy with injected headers
```

## Routing

| Gateway prefix | Target service | Note |
|---|---|---|
| `/auth/v1/` | auth-service | - |
| `/properties/` | property-service | - |
| `/vehicles/v1/` | vehicles-service | Rewrite: `/vehicles/v1/` → `/vehicles/` |
| `/finance/v1/` | finance-service | Rewrite: `/finance/v1/` → `/finance/` |
| `/billing/v1/` | billing-service | RBAC: owner only (except public paths) |
| `/notify/v1/` | notify-service | - |

## JWKS Cache

The gateway caches JWKS from Cloudflare KV for 5 minutes:

- At startup (or cache expiry): reads `jwks.json` from KV namespace
- Creates a `LocalJWKSet` with the `jose` library
- Validates JWTs with `RS256` algorithm, issuer `https://api.cuibos.ro/auth`, audience `cuibos-api`

## CORS

Allowed origins:
- `http://localhost:3000` (development)
- `https://app.cuibos.ro` (production)

Allowed headers: `Authorization`, `Content-Type`, `X-User-Id`, `X-Role`, `X-Plan`, `X-Tenant-Id`, `X-Locale`, `X-Request-Id`, `Accept-Language`.

## Cloudflare Access

The Cloudflare Access tunnel secures the connection between the Worker and the K3s cluster. The gateway attaches `CF-Access-Client-Id` and `CF-Access-Client-Secret` to every downstream request.

## JWKS Public Endpoint

```
GET /.well-known/jwks.json
```

Returns RSA public keys from KV. Cache: 5 minutes (`Cache-Control: public, max-age=300`).
