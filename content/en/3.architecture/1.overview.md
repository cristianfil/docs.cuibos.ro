# CuibOS Architecture

CuibOS is built on a **microservices** architecture with a **Cloudflare Edge Gateway** in front and asynchronous communication through **RabbitMQ**.

## Tech Stack

| Component | Technology |
|---|---|
| **Frontend** | Next.js 15, React 19, TypeScript, shadcn/ui, next-intl |
| **Gateway** | Cloudflare Worker (JavaScript, jose) |
| **Backend** | NestJS (TypeScript) per microservice |
| **ORM** | Prisma |
| **Databases** | PostgreSQL (Neon вҖ” one database per service) |
| **Event bus** | RabbitMQ (AMQP, transactional outbox) |
| **Payments** | Stripe (Checkout, Portal, Webhooks) |
| **Hosting** | K3s on Hetzner Cloud |
| **Edge** | Cloudflare Workers + Cloudflare Access |
| **CI/CD** | GitHub Actions |

## Architectural Principles

1. **Database per service** вҖ” each microservice has its own PostgreSQL database (Neon). No cross-database access.

2. **Transactional Outbox** вҖ” events are written to the `event_outbox` table in the same transaction as business data, then published asynchronously to RabbitMQ.

3. **Edge-first security** вҖ” JWT is verified at the edge (Cloudflare Worker) before the request reaches any internal service.

4. **Multi-tenant** вҖ” all data is isolated per tenant (organization). `tenantId` from JWT is the sole scoping mechanism.

5. **Event-driven** вҖ” services communicate asynchronously through events (e.g., `properties.property.created` triggers usage update in billing-service).
