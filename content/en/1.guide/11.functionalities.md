# Functionalities Reference

This document describes all features available in `app.cuibos.ro`, organized by domain. It includes data fields, validation rules, and expected behavior for each feature.

---

## 1. Authentication & Account

CuibOS uses **JWT** (JSON Web Token) based authentication validated via **JWKS** (JSON Web Key Set). Each session includes a short-lived access token and a long-lived refresh token.

### Registration

Creates a new account and an associated tenant.

::field-group

::field{name="email" type="string" required}
The user's email address. Must be unique in the system. Triggers a verification email.
::

::field{name="password" type="string" required}
Account password. Minimum 8 characters.
::

::field{name="tenantName" type="string" required}
The name of the organization / household created alongside the account.
::

::

**Post-registration behavior:**
- Account is created but marked as **unverified** until the email is confirmed
- A verification token is sent by email (expires in 24 hours)
- User is automatically authenticated and redirected to the dashboard with a verification banner

---

### Email Verification

::alert{type="warning"}
Async flow — the email may take up to 2 minutes to arrive in the inbox.
::

The verification link in the email contains a unique time-limited token. Upon access:
- Token is validated and marked as used
- Account status becomes **verified**
- Reusing an expired or already-used link returns an error

---

### Login

::field-group

::field{name="email" type="string" required}
The existing account email.
::

::field{name="password" type="string" required}
The account password.
::

::

**Response:** JWT token pair — access token (short-lived, ~15 min) + refresh token (long-lived, ~7 days).

---

### Token Refresh

The app automatically refreshes the access token using the refresh token without user interaction. Upon logout, the refresh token is revoked, invalidating the session.

---

### Forgot & Reset Password

::alert{type="warning"}
Async flow — the reset email may take up to 2 minutes to arrive.
::

::steps

### Request Reset
User submits their email on `/forgot-password`. A reset token is generated and sent by email (expires in 1 hour).

### Reset Password
On the `/reset-password` page, user enters a new password. Token is validated and marked as used.

### Login with New Password
The old password no longer works. Login with the new password succeeds.

::

---

### User Profile

::field-group

::field{name="firstName" type="string"}
User's first name. Displayed in the header and sidebar.
::

::field{name="lastName" type="string"}
User's last name.
::

::field{name="avatarUrl" type="string"}
Avatar image URL. Supports direct upload (JPEG, PNG).
::

::

---

### Password Change

Available from `/dashboard/account`. Requires the current password for confirmation.

---

### Account Deletion

::alert{type="warning"}
Available exclusively for the **owner** role. This action is irreversible and deletes the tenant and all associated data.
::

## 2. Team & Invitations

CuibOS supports multi-user collaboration within the same tenant. Members can be invited by email.

### Available Roles

| Role | Description |
|------|-------------|
| **owner** | The tenant owner. One owner per tenant. Can perform any action including account deletion and property locking. |
| **admin** | Can manage members, properties, and vehicles. Cannot delete the account or lock properties. |
| **member** | Basic access: view and record data. Cannot send invitations. |

### RBAC Matrix

| Action | owner | admin | member |
|--------|-------|-------|--------|
| Send invitation | ✓ | ✓ | ✗ |
| Revoke invitation | ✓ | ✓ | ✗ |
| Lock property | ✓ | ✗ | ✗ |
| Delete tenant account | ✓ | ✗ | ✗ |
| Create property | ✓ | ✓ | ✓ |
| Create vehicle | ✓ | ✓ | ✓ |
| Add expenses | ✓ | ✓ | ✓ |
| Read data | ✓ | ✓ | ✓ |

### Send Invitation

::field-group

::field{name="email" type="string" required}
Email address of the invited person. Can be an existing or new user.
::

::field{name="role" type="enum" required}
Role granted upon acceptance. Values: `admin`, `member`.
::

::

**Behavior:**
- Invitation has status `pending` until accepted
- Expires after 7 days if not accepted
- Can be resent or revoked by owner or admin
- If the email has no account upon acceptance, the user is directed to register

## 5. Finance — Budgets

The budgets module allows tracking income and expenses over defined periods, optionally linked to properties or vehicles.

### Budget Scopes

| Scope | Description |
|-------|-------------|
| `tenant` | Shared budget at tenant level, visible to all members |
| `personal` | Personal budget, visible only to the current user |

A budget can additionally be linked to a property (`propertyId`) or a vehicle (`vehicleId`).

### Budget Fields

::field-group

::field{name="currency" type="string" required}
Budget currency (e.g., `RON`, `EUR`, `USD`).
::

::field{name="periodStart" type="date" required}
First day of the budget period.
::

::field{name="periodEnd" type="date" required}
Last day of the budget period. Must be after `periodStart`.
::

::field{name="scope" type="enum" required}
Budget visibility. Values: `tenant`, `personal`.
::

::field{name="name" type="string"}
Optional budget name.
::

::field{name="propertyId" type="uuid"}
Links the budget to a specific property.
::

::field{name="vehicleId" type="uuid"}
Links the budget to a specific vehicle.
::

::

### Budget Strategies

| Strategy | Description |
|----------|-------------|
| `traditional` | Track expenses vs. income. Simplest usage mode. |
| `zero_based` | Every monetary unit is allocated. Planned amount must equal total budget. |
| `envelope` | Money is divided into category "envelopes" with fixed limits. |
| `goal_based` | Oriented toward reaching a savings goal. |

---

### Budget Line Items

::field-group

::field{name="category" type="string" required}
Expense / income category.
::

::field{name="amount" type="number" required}
Planned or actual amount.
::

::field{name="currency" type="string"}
Line item currency (can differ from budget currency; conversion is done automatically via BNR rates).
::

::field{name="planningStatus" type="enum"}
Line type. Values: `planned` (expected expense), `unplanned` (unexpected expense).
::

::field{name="notes" type="string"}
Additional notes.
::

::

---

### Exchange Rates (BNR)

CuibOS automatically fetches exchange rates from the **National Bank of Romania (BNR)**. When adding a budget line item in a different currency than the budget's, the system applies the BNR rate at that date and displays the rate used.

## 8. Reminders & Notifications

The reminder system automatically tracks important dates and sends email notifications before they are due.

### Reminder Domains

| Domain | Data Source | Default Offsets |
|--------|-------------|----------------|
| `rca` | Vehicles — `rcaExpiryDate` field | 60, 30, 7, 1, 0 days |
| `itp` | Vehicles — `itpExpiryDate` field | 60, 30, 7, 1, 0 days |
| `ro_vignette` | Vehicles — `vignetteExpiryDate` field | 30, 7, 1, 0 days |
| `bill_due` | Bills — `dueDate` field | 14, 7, 1, 0 days |
| `installment_due` | Installments — due date | 14, 7, 1, 0 days |
| `maintenance` | Vehicle maintenance — due date/km | Configured per item |

### Reminder Settings

::field-group

::field{name="domain" type="enum" required}
Reminder domain (e.g., `rca`, `bill_due`).
::

::field{name="enabled" type="boolean"}
Enables or disables all reminders in this domain.
::

::field{name="offsetDays" type="integer[]"}
List of warning days before due date. Changing the list cancels old occurrences and creates new ones.
::

::field{name="timezone" type="string"}
Timezone for calculating send times. Default: `Europe/Bucharest`.
::

::

### Reminder Lifecycle

::steps

### Materialization
Each cron cycle, the reminder-service calculates upcoming occurrences (30, 7, 1 days before due date) and creates them in the database with `pending` status.

### Dispatch
The dispatch cron (every ~10 seconds) picks up `pending` occurrences with `fireAt` ≤ current time and publishes them to the AMQP queue.

### Email Delivery
The notify-service consumes the event and sends the email in the tenant's language (RO or EN).

### Completion
The occurrence is marked as `sent`. On error, it is marked `failed` (up to 3 retries) or `dead` (after 3 failures).

::

::alert{type="warning"}
Reminders are sent asynchronously. The normal delay from trigger to receipt is **under 1 minute** in production.
::

## 9. Dashboard

The home page (`/dashboard`) provides a centralized view of the household's status.

### KPI Widgets

| Widget | Description |
|--------|-------------|
| Properties | Total number of active properties in the tenant |
| Vehicles | Total number of active vehicles in the tenant |
| Current Budget | Summary of the current month's budget (actual vs. planned) |
| Net Balance | Aggregated balance of all active budgets, converted to the base currency |

### Attention Items

The "Attention" section proactively surfaces issues that require action:

| Type | Display Condition |
|------|------------------|
| Budget at risk | Actual spending exceeds 80% of planned amount |
| Missing utilities | Active property with no configured utilities |
| Compliance due | Vehicle with RCA / ITP / vignette expiring within the warning window |
| Overdue bill | Bill with a past due date and unpaid |
| Maintenance due | Maintenance item with exceeded date or odometer |

### Recent Activity Feed

Lists the latest actions performed in the tenant, ordered in reverse chronological order, with:
- Action title (e.g., "New property added")
- Affected entity (e.g., "Floreasca Apartment")
- Relative timestamp (e.g., "5 minutes ago")

### Currency Conversion

Dashboard totals are calculated in the tenant's base currency. Amounts in other currencies are automatically converted using the BNR rate at the current date or the most recently available date.
