# Utilities & Meters

Each property can have multiple utilities configured, and each utility can have one or more meters.

## Utility types

| Type | Description |
|---|---|
| `electricity` | Electric power |
| `gas` | Natural gas |
| `water` | Water (cold / hot) |
| `heating` | District heating |

## Configuring utilities

From the property page → **Configuration** → **Utilities**:

::steps

### Add a utility

Select the type (e.g., electricity) and fill in the provider information:
- **Provider** — company name (e.g., Enel, E.ON)
- **Contract no.** — contract number
- **Start / end day** — the month interval for submitting readings

### Add meters

Each utility can have multiple meters:
- **Label** — meter name (e.g., "Day meter", "Night meter")
- **Meter number** — meter serial number
- **Unit** — unit of measurement (kWh, m³, Gcal)

### Save configuration

The configuration is saved in full (utilities + meters) through a single `PUT` request.

::

## Recording readings

From the property page → **Consumption**:

1. Select the desired meter
2. Enter the **reading value** and **reading date**
3. Optionally mark if it was **submitted to the provider**
4. Save

::alert{type="info"}
Consumption cycles are calculated automatically based on consecutive readings. If you correct an old reading, you can manually recalculate cycles.
::

## Viewing consumption

The consumption page displays:
- **Recent readings** — table with latest readings per meter
- **Cycles** — calculated consumption over periods (months)
- **Consumption chart** — chart-friendly time series

## Index table

The index table provides a monthly aggregated view of consumption per utility, ideal for quick comparisons. Accessible from the property → **Index table**.
