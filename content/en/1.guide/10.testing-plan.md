# Functional Testing Plan

This document describes all manual test scenarios for `app.cuibos.ro`, covering the 7 backend microservices (auth, property, finance, vehicles, billing, notify, reminder).

## Legend

| Symbol | Meaning |
|--------|---------|
| **P0** | Critical — must pass before any release |
| **P1** | High — must pass before production |
| **P2** | Medium — acceptable to defer one cycle |
| **[EMAIL]** | Async flow — requires inbox access |
| **[STRIPE]** | Requires Stripe test mode |
| **[OWNER]** | Owner role only |
| **[ADMIN+]** | Owner or admin role |

---

## Test Environment Setup

### Prerequisites

- Browser: Chrome (latest), with DevTools available
- Three email inboxes (use `+` aliases):
  - `qa-owner@domain.com` — primary tester / tenant owner
  - `qa-member@domain.com` — invited team member (admin role)
  - `qa-invite@domain.com` — new-user invitation target
- Stripe test-mode dashboard access (for billing suite)
- Two browser profiles or one incognito window for multi-user scenarios

### Test Data Baseline

| Entity | Key Fields |
|--------|-----------|
| **Account A** (owner) | qa-owner@domain.com, verified email, tenant: "QA Test Org" |
