# Auth Service

Prefix: `/auth/v1`

The authentication service manages accounts, JWT sessions, email verification, team invitations and passwords.

## Registration & Authentication

### POST /auth/v1/register

Creates a new account (free plan).

::field-group
  ::field{name="email" type="string" required}
  Email address
  ::
  ::field{name="password" type="string" required}
  Password (minimum 8 characters)
  ::
  ::field{name="name" type="string"}
  Display name
  ::
  ::field{name="locale" type="string"}
  Preferred language (`ro` / `en`)
  ::
::

---

### POST /auth/v1/register-paid

Creates an account associated with an already-paid Stripe checkout.

::field-group
  ::field{name="email" type="string" required}
  Email address (must match the Stripe checkout email)
  ::
  ::field{name="password" type="string" required}
  Password
  ::
  ::field{name="name" type="string"}
  Display name
  ::
  ::field{name="checkoutSessionId" type="string" required}
  Stripe checkout session ID
  ::
::

---

### POST /auth/v1/login

Authenticate with email and password. Returns access token and sets refresh token as httpOnly cookie.

::field-group
  ::field{name="email" type="string" required}
  Email address
  ::
  ::field{name="password" type="string" required}
  Password
  ::
::

**Response:**
```json
{
  "accessToken": "eyJhbGciOi...",
  "user": {
    "id": "uuid",
    "email": "user@example.com",
    "name": "Cristian",
    "role": "owner",
    "plan": "pro",
    "tenantId": "uuid"
  }
}
```

---

### POST /auth/v1/logout

Invalidates the current session and clears the refresh token cookie.

---

### POST /auth/v1/refresh

Renews the access token using the refresh token from the cookie.

**Response:**
```json
{
  "accessToken": "eyJhbGciOi..."
}
```

---

### POST /auth/v1/introspect

Verifies and decodes an access token. Useful for client-side validation.

---

## User Profile

### GET /auth/v1/me

🔒 **Requires JWT**

Returns the authenticated user's profile.

---

### POST /auth/v1/update-profile

🔒 **Requires JWT**

Updates the user profile (name, preferences).

---

### POST /auth/v1/delete-account

🔒 **Requires JWT**

Deletes the user account and all associated data.

---

---

### GET /auth/v1/invitations

🔒 **Requires JWT**

Returns the list of invitations sent by the current tenant.

---

### POST /auth/v1/invitations/:id/resend

🔒 **Requires JWT**

Resends the invitation with the specified ID.

---

### POST /auth/v1/invitations/:id/revoke

🔒 **Requires JWT**

Revokes (cancels) the invitation with the specified ID.

## JWKS

### GET /auth/v1/jwks.json

Returns the public key set (JWKS) used for JWT verification. Used internally by the gateway.

```json
{
  "keys": [
    {
      "kty": "RSA",
      "kid": "...",
      "n": "...",
      "e": "AQAB",
      "alg": "RS256",
      "use": "sig"
    }
  ]
}
```
