# API Overview

CuibOS exposes a REST API through a **Cloudflare Edge Gateway** that routes requests to internal microservices.

## Base URL

```
https://api.cuibos.ro
```

## Authentication

All protected endpoints require a **JWT Bearer token** in the `Authorization` header:

```http
Authorization: Bearer <access_token>
```

The token is obtained via the `/auth/v1/login` endpoint and has a standard 15-minute lifetime. Renewal is done via `/auth/v1/refresh` with a refresh token (httpOnly cookie).

## Standard headers

| Header | Required | Description |
|---|---|---|
| `Authorization` | Yes (protected) | `Bearer <JWT>` |
| `Content-Type` | Yes (POST/PUT/PATCH) | `application/json` |
| `X-Request-Id` | No | Unique ID for request correlation (auto-generated if missing) |
| `X-Locale` | No | Preferred language: `ro` or `en` |
| `Accept-Language` | No | Alternative for locale |

## Headers injected by gateway

The Cloudflare gateway extracts claims from the JWT and attaches them as downstream headers:

| Header | Source | Description |
|---|---|---|
| `X-User-Id` | `payload.sub` | User ID |
| `X-User-Email` | `payload.email` | User email |
| `X-Tenant-Id` | `payload.tenantId` | Tenant ID |
| `X-Role` | `payload.role` | Role: owner, admin, viewer |
| `X-Plan` | `payload.plan` | Plan: free, pro, pro_plus |

## Service routing

| Gateway prefix | Internal service |
|---|---|
| `/auth/v1/` | auth-service |
| `/properties/` | property-service |
| `/vehicles/v1/` | vehicles-service |
| `/finance/v1/` | finance-service |
| `/billing/v1/` | billing-service |
| `/notify/v1/` | notify-service |

## Public endpoints

The following endpoints **do not require a JWT**:

- `POST /auth/v1/login`
- `POST /auth/v1/register`
- `POST /auth/v1/register-paid`
- `POST /auth/v1/verify-email`
- `POST /auth/v1/forgot-password`
- `POST /auth/v1/reset-password`
- `POST /auth/v1/refresh`
- `POST /auth/v1/introspect`
- `POST /auth/v1/accept-invite`
- `GET /auth/v1/jwks.json`
- `POST /billing/v1/public/checkout`
- `GET /billing/v1/public/checkout-session/:sessionId`
- `POST /billing/v1/webhooks/stripe`

## Error codes

| Code | Description |
|---|---|
| `401` | Missing or expired token |
| `403` | Insufficient role |
| `422` | Validation error (invalid fields) |
| `404` | Resource not found |
| `409` | Conflict (duplicate resource) |
| `500` | Internal server error |

## Response conventions

- **IDs** are UUID v4
- **Dates** are ISO 8601 (`2025-01-15T10:30:00Z`) or `YYYY-MM-DD` for date-only fields
- **Amounts** are serialized as strings (e.g., `"129.50"`) for decimal precision
- **Enums** are lowercase strings
