# Billing Service

Prefix: `/billing/v1`

The billing service manages subscriptions through Stripe integration.

## Checkout

### POST /billing/v1/checkout

🔒 **Requires JWT** (owner only)

Creates a Stripe Checkout session for plan upgrade.

::field-group
  ::field{name="planName" type="string" required}
  Plan name: `pro` or `pro_plus`
  ::
::

**Response:**
```json
{
  "url": "https://checkout.stripe.com/c/pay/..."
}
```

---

### POST /billing/v1/public/checkout

Creates a Stripe Checkout session **without authentication** (for the paid registration flow).

::field-group
  ::field{name="planName" type="string" required}
  Plan name: `pro` or `pro_plus`
  ::
  ::field{name="email" type="string"}
  Customer email (optional)
  ::
::

---

### GET /billing/v1/public/checkout-session/:sessionId

Returns checkout session details (public, for post-payment verification).

---

## Stripe Portal

### POST /billing/v1/portal

🔒 **Requires JWT** (owner only)

Generates a link to the Stripe portal for managing the subscription.

**Response:**
```json
{
  "url": "https://billing.stripe.com/p/session/..."
}
```

---

## Subscription

### GET /billing/v1/subscription

🔒 Returns current subscription details.

**Response:**
```json
{
  "plan": "pro",
  "status": "active",
  "currentPeriodEnd": "2025-02-15T00:00:00Z",
  "cancelAtPeriodEnd": false
}
```

---

## Subscription Invoices

### GET /billing/v1/invoices

🔒 Returns the tenant's Stripe invoices.

---

## Webhooks

### POST /billing/v1/webhooks/stripe

Endpoint for Stripe webhooks. **Does not require JWT** — authenticated via Stripe signature.

Handled events:
- `checkout.session.completed` — subscription activation
- `customer.subscription.updated` — plan changes
- `customer.subscription.deleted` — subscription cancellation
- `invoice.payment_succeeded` — successful payment
- `invoice.payment_failed` — failed payment
