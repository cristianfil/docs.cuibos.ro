## Stack tehnologic

| ComponentДғ | Tehnologie |
|---|---|
| **Frontend** | Next.js 15, React 19, TypeScript, shadcn/ui, next-intl |
| **Gateway** | Cloudflare Worker (JavaScript, jose) |
| **Backend** | NestJS (TypeScript) per microserviciu |
| **ORM** | Prisma |
| **Baze de date** | PostgreSQL (Neon вҖ” o bazДғ per serviciu) |
| **Event bus** | RabbitMQ (AMQP, transactional outbox) |
| **PlДғИӣi** | Stripe (Checkout, Portal, Webhooks) |
| **Hosting** | K3s pe Hetzner Cloud |
| **Edge** | Cloudflare Workers + Cloudflare Access |
| **CI/CD** | GitHub Actions |

## Principii arhitecturale

1. **Database per service** вҖ” fiecare microserviciu are propria bazДғ de date PostgreSQL (Neon). Nu existДғ acces cross-database.

2. **Transactional Outbox** вҖ” evenimentele sunt scrise Г®n tabela `event_outbox` Г®n aceeaИҷi tranzacИӣie cu datele de business, apoi publicate asincron pe RabbitMQ.

3. **Edge-first security** вҖ” JWT-ul este verificat la edge (Cloudflare Worker) Г®nainte ca request-ul sДғ ajungДғ la oricare serviciu intern.

4. **Multi-tenant** вҖ” toate datele sunt izolate per tenant (organizaИӣie). `tenantId` din JWT este singurul mecanism de scoping.

5. **Event-driven** вҖ” serviciile comunicДғ asincron prin evenimente (ex: `properties.property.created` declanИҷeazДғ actualizarea usage-ului Г®n billing-service).
