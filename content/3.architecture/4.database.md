# Baze de date

CuibOS folosește pattern-ul **database per service** — fiecare microserviciu are propria bază de date PostgreSQL izolată, gestionată prin **Neon** (serverless Postgres).

## Schema per serviciu

### auth-service

Tabele principale:
- `users` — conturi utilizator
- `tenants` — organizații (conturi de familie)
- `tenant_members` — asocierea utilizatorilor cu tenant-ul
- `invitations` — invitații de echipă
- `refresh_tokens` — sesiuni active
- `verification_tokens` — token-uri email
- `password_reset_tokens` — token-uri resetare parolă

### property-service

Tabele principale:
- `properties` — proprietăți imobiliare
- `property_users` — acces utilizatori la proprietăți (landlord, tenant, manager, viewer)
- `property_utilities` — utilități (electricity, gas, water, heating)
- `utility_meters` — contoare individuale
- `meter_readings` — citiri contoare
- `meter_cycles` — cicluri consum calculate
- `property_services` — servicii recurente
- `service_catalog_items` — catalog servicii predefinite
- `rental_agreements` — contracte de închiriere
- `event_outbox` — transactional outbox

### vehicles-service

Tabele principale:
- `vehicles` — vehicule
- `vehicle_expenses` — cheltuieli vehicule
- `odometer_readings` — citiri kilometraj
- `maintenance_items` — planificare mentenanță
- `maintenance_logs` — jurnal mentenanță
- `event_outbox` — transactional outbox

### finance-service

Tabele principale:
- `budgets` — bugete familiale
- `budget_lines` — linii buget
- `bills` — facturi
- `installments` — rate
- `installment_payments` — plăți rate

### billing-service

Tabele principale:
- `subscriptions` — abonamente Stripe
- `signup_checkouts` — sesiuni checkout pre-înregistrare
- `property_usage_states` — tracking proprietăți per tenant
- `vehicle_usage_states` — tracking vehicule per tenant
- `usage_sync_jobs` — sincronizare usage cu Stripe

## Neon (Serverless PostgreSQL)

Fiecare serviciu se conectează la un branch Neon separat:
- **Connection pooling** — Neon oferă PgBouncer nativ
- **Autoscaling** — compute se scalează automat
- **Branching** — posibilitate de branch-uri pentru teste

## Prisma ORM

Toate serviciile folosesc **Prisma** ca ORM:

- `prisma/schema.prisma` — definirea modelelor
- `prisma migrate dev` — migrări în dezvoltare
- `prisma migrate deploy` — migrări în producție
- `prisma generate` — generarea clientului TypeScript

### Convenții

- **UUID-uri** pentru toate primary keys
- **Soft delete** cu `deletedAt` (nullable timestamp)
- **Timestamps** automatice: `createdAt`, `updatedAt`
- **Enum-uri** definite în Prisma schema, mapate la PostgreSQL native enums
- **Decimal** stocat ca `Decimal` Prisma → serializat ca string în JSON
