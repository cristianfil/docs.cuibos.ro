# Evenimente

CuibOS folosește o arhitectură **event-driven** cu **transactional outbox** pentru comunicarea între microservicii.

## Pattern: Transactional Outbox

```
┌──────────────────────────────────┐
│         Tranzacție DB            │
│                                  │
│  1. INSERT/UPDATE date business  │
│  2. INSERT eveniment în outbox   │
│                                  │
│         COMMIT                   │
└──────────────┬───────────────────┘
               │
               ▼
┌──────────────────────────────────┐
│     Outbox Poller (async)        │
│                                  │
│  3. Citește evenimente nepublicate│
│  4. Publică pe RabbitMQ          │
│  5. Marchează ca publicate       │
└──────────────────────────────────┘
```

Avantaje:
- **Atomicitate** — datele și evenimentul se salvează în aceeași tranzacție
- **Fiabilitate** — dacă RabbitMQ e indisponibil, evenimentele rămân în outbox
- **Idempotență** — consumatorii pot procesa același eveniment de mai multe ori

## Tabela event_outbox

Fiecare serviciu care produce evenimente are o tabelă `event_outbox`:

| Coloană | Tip | Descriere |
|---|---|---|
| `id` | UUID | PK |
| `routingKey` | string | Cheia de rutare (ex: `properties.property.created`) |
| `payload` | JSON | Corpul evenimentului |
| `tenantId` | UUID | Tenant-ul sursă |
| `correlationId` | string | ID-ul cererii originale |
| `publishedAt` | timestamp | Null până la publicare |
| `createdAt` | timestamp | Momentul creării |

## Exchange RabbitMQ

Toate evenimentele se publică pe exchange-ul `events` (topic exchange). Consumatorii se abonează folosind routing keys cu wildcards.

## Categorii de evenimente

### Property Events

| Routing key | Payload | Produs de |
|---|---|---|
| `properties.property.created` | `{tenantId, propertyId, name, propertyType, locked}` | property-service |
| `properties.property.updated` | `{tenantId, propertyId, name, propertyType, locked}` | property-service |
| `properties.property.deleted` | `{tenantId, propertyId, deletedAt}` | property-service |
| `properties.property_user.activated` | `{tenantId, propertyId, userId, role}` | property-service |
| `properties.property_user.revoked` | `{tenantId, propertyId, userId}` | property-service |

**Consumatori:** billing-service (usage tracking)

### Vehicle Events

| Routing key | Payload | Produs de |
|---|---|---|
| `vehicles.vehicle.created` | `{tenantId, vehicleId}` | vehicles-service |
| `vehicles.vehicle.updated` | `{tenantId, vehicleId}` | vehicles-service |
| `vehicles.vehicle.deleted` | `{tenantId, vehicleId}` | vehicles-service |

**Consumatori:** billing-service (usage tracking)

### Auth Events

| Routing key | Payload | Produs de |
|---|---|---|
| `auth.tenant.plan_changed` | `{tenantId, plan, previousPlan}` | auth-service |
| `auth.user.registered` | `{tenantId, userId, email}` | auth-service |

**Consumatori:** billing-service, property-service

## Fluxuri inter-servicii

### Creare proprietate → billing usage

```
property-service                    billing-service
     │                                   │
     │  properties.property.created      │
     ├──────────────────────────────────▶│
     │                                   │
     │                    Actualizare PropertyUsageState
     │                    Recompute subscription usage
     │                                   │
```

### Înregistrare cu plan plătit

```
Stripe → billing-service (webhook)
         │
         │  Creare SignupCheckout record
         │
         ▼
app → auth-service (register-paid)
      │
      │  Claim SignupCheckout
      │  Asociere Stripe customer + subscription cu tenant
      │
      ▼
auth-service → billing-service (event: tenant.plan_changed)
```
