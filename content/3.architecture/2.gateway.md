# Edge Gateway

Gateway-ul CuibOS este un **Cloudflare Worker** care rulează la edge (cel mai apropiat datacenter de utilizator).

## Responsabilități

1. **Rutarea cererilor** — direcționează prefix-urile URL către serviciile interne
2. **Verificarea JWT** — validează token-ul cu JWKS din Cloudflare KV
3. **Injectarea claim-urilor** — extrage date din JWT și le atașează ca header-e
4. **RBAC** — verifică rolul utilizatorului pentru anumite rute
5. **CORS** — gestionează preflight-urile și header-ele cross-origin
6. **Locale** — propagă limba preferată din `X-Locale` sau `Accept-Language`

## Fluxul unei cereri

```
Client
  │
  ├─ OPTIONS (preflight) → 204 cu CORS headers
  │
  └─ GET/POST/... cu Authorization: Bearer <JWT>
        │
        ├─ Public path? → proxy direct la serviciu
        │
        └─ Protected path:
              ├─ Lipsă token → 401
              ├─ Token invalid/expirat → 401
              ├─ Rol insuficient → 403
              └─ Valid → proxy cu header-e injectate
```

## Rutare

| Prefix gateway | Serviciu destinație | Notă |
|---|---|---|
| `/auth/v1/` | auth-service | - |
| `/properties/` | property-service | - |
| `/vehicles/v1/` | vehicles-service | Rewrite: `/vehicles/v1/` → `/vehicles/` |
| `/finance/v1/` | finance-service | Rewrite: `/finance/v1/` → `/finance/` |
| `/billing/v1/` | billing-service | RBAC: owner only (cu excepția public paths) |
| `/notify/v1/` | notify-service | - |

## JWKS Cache

Gateway-ul cachează JWKS-ul din Cloudflare KV pentru 5 minute:

- La pornire (sau cache expirat): citește `jwks.json` din KV namespace
- Creează un `LocalJWKSet` cu `jose` library
- Validează JWT-urile cu algoritm `RS256`, issuer `https://api.cuibos.ro/auth`, audience `cuibos-api`

## CORS

Origini permise:
- `http://localhost:3000` (dezvoltare)
- `https://app.cuibos.ro` (producție)

Header-e permise: `Authorization`, `Content-Type`, `X-User-Id`, `X-Role`, `X-Plan`, `X-Tenant-Id`, `X-Locale`, `X-Request-Id`, `Accept-Language`.

## Cloudflare Access

Tunelul Cloudflare Access securizează conexiunea între Worker și clusterul K3s. Gateway-ul atașează `CF-Access-Client-Id` și `CF-Access-Client-Secret` la fiecare cerere downstream.

## JWKS Public Endpoint

```
GET /.well-known/jwks.json
```

Returnează cheile publice RSA din KV. Cache: 5 minute (`Cache-Control: public, max-age=300`).
