# Referință funcționalități

Acest document descrie toate funcționalitățile disponibile în `app.cuibos.ro`, organizate pe domenii. Include câmpurile de date, regulile de validare și comportamentul așteptat pentru fiecare funcție.

---

## 1. Autentificare & cont

CuibOS folosește autentificare bazată pe **JWT** (JSON Web Tokens) cu validare prin **JWKS** (JSON Web Key Set). Fiecare sesiune include un access token de scurtă durată și un refresh token de lungă durată.

### Înregistrare

Crearea unui cont nou și a unui tenant asociat.

::field-group

::field{name="email" type="string" required}
Adresa de email a utilizatorului. Trebuie să fie unică în sistem. Declanșează trimiterea unui email de verificare.
::

::field{name="password" type="string" required}
Parola contului. Minim 8 caractere.
::

::field{name="tenantName" type="string" required}
Numele organizației / gospodăriei create odată cu contul.
::

::

**Comportament post-înregistrare:**
- Contul este creat dar marcat ca **neverificat** până la confirmarea email-ului
- Un token de verificare este trimis prin email (expiră în 24 ore)
- Utilizatorul este autentificat automat și redirecționat la dashboard cu un banner de verificare

---

### Verificare email

::alert{type="warning"}
Flux asincron — email-ul poate dura până la 2 minute să ajungă în inbox.
::

Linkul de verificare din email conține un token unic cu durată de viață limitată. La accesare:
- Token-ul este validat și marcat ca folosit
- Contul capătă statusul **verificat**
- Refolosirea unui link expirat sau deja utilizat returnează eroare

---

### Autentificare

::field-group

::field{name="email" type="string" required}
Email-ul contului existent.
::

::field{name="password" type="string" required}
Parola contului.
::

::

**Răspuns:** Pereche de tokeni JWT — access token (scurt, ~15 min) + refresh token (lung, ~7 zile).

---

### Refresh token

Aplicația reîmprospătează automat access token-ul folosind refresh token-ul fără intervenția utilizatorului. La deconectare, refresh token-ul este revocat, invalidând sesiunea.

---

### Recuperare & resetare parolă

::alert{type="warning"}
Flux asincron — email-ul de resetare poate dura până la 2 minute.
::

::steps

### Solicitare resetare
Utilizatorul trimite adresa de email pe `/forgot-password`. Un token de resetare este generat și trimis prin email (expirat în 1 oră).

### Resetare parolă
Pe pagina `/reset-password`, utilizatorul introduce parola nouă. Token-ul se validează și se marchează ca utilizat.

### Autentificare cu parola nouă
Parola veche nu mai funcționează. Autentificarea cu parola nouă reușește.

::

---

### Profil utilizator

::field-group

::field{name="firstName" type="string"}
Prenumele utilizatorului. Afișat în header și sidebar.
::

::field{name="lastName" type="string"}
Numele de familie al utilizatorului.
::

::field{name="avatarUrl" type="string"}
URL-ul imaginii de avatar. Suportă upload direct (JPEG, PNG).
::

::

---

### Schimbare parolă

Disponibilă din `/dashboard/account`. Necesită parola curentă pentru confirmare.

---

### Ștergere cont

::alert{type="warning"}
Disponibil exclusiv pentru rolul **owner**. Acțiunea este ireversibilă și șterge tenantul și toate datele asociate.
::

## 2. Echipă & invitații

CuibOS suportă colaborare multi-utilizator în cadrul aceluiași tenant. Membrii pot fi invitați prin email.

### Roluri disponibile

| Rol | Descriere |
|-----|-----------|
| **owner** | Proprietarul tenantului. Un singur owner per tenant. Poate face orice acțiune inclusiv ștergerea contului și blocarea proprietăților. |
| **admin** | Poate administra membrii, proprietățile și vehiculele. Nu poate șterge contul sau bloca proprietăți. |
| **member** | Acces de bază: vizualizare și înregistrare date. Nu poate trimite invitații. |

### Matrice RBAC

| Acțiune | owner | admin | member |
|---------|-------|-------|--------|
| Trimitere invitație | ✓ | ✓ | ✗ |
| Revocare invitație | ✓ | ✓ | ✗ |
| Blocare proprietate | ✓ | ✗ | ✗ |
| Ștergere cont tenant | ✓ | ✗ | ✗ |
| Creare proprietate | ✓ | ✓ | ✓ |
| Creare vehicul | ✓ | ✓ | ✓ |
| Adăugare cheltuieli | ✓ | ✓ | ✓ |
| Citire date | ✓ | ✓ | ✓ |

### Trimitere invitație

::field-group

::field{name="email" type="string" required}
Adresa de email a persoanei invitate. Poate fi un utilizator existent sau nou.
::

::field{name="role" type="enum" required}
Rolul acordat după acceptare. Valori: `admin`, `member`.
::

::

**Comportament:**
- Invitația are status `pending` până la acceptare
- Expiră după 7 zile dacă nu este acceptată
- Poate fi retrimisă sau revocată de owner sau admin
- La acceptare, dacă email-ul nu are cont, utilizatorul este direcționat să se înregistreze

## 3. Proprietăți

Modulul central al platformei. O proprietate reprezintă orice imobil administrat: casă, apartament, teren etc.

### Tipuri de proprietăți

| Tip | Descriere |
|-----|-----------|
| `apartment` | Apartament |
| `house` | Casă |
| `studio` | Garsonieră |
| `office` | Birou / spațiu comercial |
| `land` | Teren |

### Statusuri proprietate

| Status | Descriere |
|--------|-----------|
| `active` | Proprietate activă (default) |
| `rented` | Proprietate închiriată |
| `inactive` | Proprietate inactivă / scoasă din uz |

### Câmpuri proprietate

::field-group

::field{name="name" type="string" required}
Numele proprietății. Maxim 255 de caractere.
::

::field{name="type" type="enum" required}
Tipul imobilului. Valori: `apartment`, `house`, `studio`, `office`, `land`.
::

::field{name="status" type="enum"}
Statusul curent. Default: `active`.
::

::field{name="address" type="string"}
Adresa stradală.
::

::field{name="city" type="string"}
Orașul.
::

::field{name="county" type="string"}
Județul.
::

::field{name="postalCode" type="string"}
Codul poștal.
::

::field{name="surfaceArea" type="number"}
Suprafața în m². Valoare pozitivă.
::

::field{name="rooms" type="integer"}
Numărul de camere.
::

::field{name="bathrooms" type="integer"}
Numărul de băi.
::

::field{name="description" type="string"}
Descriere liberă a proprietății.
::

::

### Blocare proprietate

::alert{type="info"}
Disponibil exclusiv pentru rolul **owner**.
::

O proprietate blocată nu poate fi editată de alți membri. Util pentru proprietăți cu date finale (ex: contracte încheiate).

---

### Configurare proprietate

Accesibilă din `/dashboard/properties/[id]/config`. Permite configurarea utilităților și serviciilor asociate proprietății.

#### Utilități contorizate

Utilitățile contorizate urmăresc consumul prin citiri periodice de contor.

| Tip utilitate | Unitate implicită |
|---------------|------------------|
| `electricity` | kWh |
| `gas` | m³ |
| `water` | m³ |
| `heating` | Gcal |

::field-group

::field{name="type" type="enum" required}
Tipul utilității. Valori: `electricity`, `gas`, `water`, `heating`.
::

::field{name="providerName" type="string"}
Numele furnizorului (ex: E.ON, Engie).
::

::field{name="contractNumber" type="string"}
Numărul contractului cu furnizorul.
::

::field{name="readingSubmissionWindowStart" type="integer"}
Prima zi din lună când se pot trimite citiri (1–31).
::

::field{name="readingSubmissionWindowEnd" type="integer"}
Ultima zi din lună pentru trimitere citiri (1–31).
::

::

#### Contoare

Fiecare utilitate poate avea unul sau mai multe contoare.

::field-group

::field{name="label" type="string" required}
Eticheta contorului (ex: „Principal", „Subsol").
::

::field{name="unit" type="string" required}
Unitatea de măsură (ex: `kWh`, `m³`).
::

::field{name="serialNumber" type="string"}
Seria contorului fizic.
::

::

#### Citiri contor

::field-group

::field{name="value" type="number" required}
Valoarea citită de pe contor. Trebuie să fie mai mare sau egală cu citirea anterioară.
::

::field{name="readAt" type="date" required}
Data la care s-a efectuat citirea.
::

::

**Comportament:** La fiecare nouă citire, sistemul calculează automat un **ciclu de consum** față de citirea anterioară. Ciclurile pot fi recalculate manual dacă este necesar.

---

#### Servicii non-contorizate

Serviciile sunt cheltuieli recurente asociate proprietății care nu necesită citiri (ex: internet, gunoi, asigurare).

::field-group

::field{name="type" type="string" required}
Tipul serviciului (ex: `internet`, `trash`, `insurance`).
::

::field{name="label" type="string" required}
Eticheta afișată (ex: „RCS Internet").
::

::field{name="providerName" type="string"}
Numele furnizorului serviciului.
::

::field{name="cadence" type="enum" required}
Frecvența de facturare. Valori: `monthly`, `quarterly`, `yearly`, `custom`.
::

::field{name="recurrenceUnit" type="enum"}
Unitatea de recurență dacă cadence=`custom`. Valori: `day`, `week`, `month`, `year`.
::

::field{name="recurrenceEvery" type="integer"}
Frecvența numerică (ex: `3` = la fiecare 3 luni).
::

::field{name="invoiceAmount" type="number"}
Suma obișnuită a facturii (opțional, pentru estimări).
::

::

## 5. Finanțe — Bugete

Modulul de bugete permite urmărirea veniturilor și cheltuielilor pe perioade definite, legate opțional de proprietăți sau vehicule.

### Scopuri buget

| Scope | Descriere |
|-------|-----------|
| `tenant` | Buget comun la nivel de tenant, vizibil tuturor membrilor |
| `personal` | Buget personal, vizibil doar utilizatorului curent |

Un buget poate fi legat suplimentar de o proprietate (`propertyId`) sau de un vehicul (`vehicleId`).

### Câmpuri buget

::field-group

::field{name="currency" type="string" required}
Moneda bugetului (ex: `RON`, `EUR`, `USD`).
::

::field{name="periodStart" type="date" required}
Prima zi a perioadei bugetare.
::

::field{name="periodEnd" type="date" required}
Ultima zi a perioadei bugetare. Trebuie să fie după `periodStart`.
::

::field{name="scope" type="enum" required}
Vizibilitatea bugetului. Valori: `tenant`, `personal`.
::

::field{name="name" type="string"}
Denumire opțională a bugetului.
::

::field{name="propertyId" type="uuid"}
Leagă bugetul de o proprietate specifică.
::

::field{name="vehicleId" type="uuid"}
Leagă bugetul de un vehicul specific.
::

::

### Strategii buget

| Strategie | Descriere |
|-----------|-----------|
| `traditional` | Urmărire cheltuieli vs. venituri. Cel mai simplu mod de utilizare. |
| `zero_based` | Fiecare leu este alocat. Suma planificată trebuie să egaleze bugetul total. |
| `envelope` | Banii sunt împărțiți în „plicuri" pe categorii cu limite fixe. |
| `goal_based` | Orientat spre atingerea unui obiectiv de economisire. |

---

### Linii buget

::field-group

::field{name="category" type="string" required}
Categoria cheltuielii / venitului.
::

::field{name="amount" type="number" required}
Suma planificată sau reală.
::

::field{name="currency" type="string"}
Moneda liniei (poate diferi de moneda bugetului; conversia se face automat prin cursul BNR).
::

::field{name="planningStatus" type="enum"}
Tipul liniei. Valori: `planned` (cheltuială prevăzută), `unplanned` (cheltuială neprevăzută).
::

::field{name="notes" type="string"}
Observații suplimentare.
::

::

---

### Cursuri valutare (BNR)

CuibOS obține automat cursurile valutare de la **Banca Națională a României (BNR)**. La adăugarea unei linii buget în altă monedă decât cea a bugetului, sistemul aplică cursul BNR de la data respectivă și afișează cursul utilizat.

## 7. Facturare & Abonament

CuibOS oferă trei planuri de abonament cu limite și funcționalități diferite.

### Planuri disponibile

::alert{type="info"}
Toate prețurile sunt exprimate fără TVA.
::

| Caracteristică | Free | Pro | Pro+ |
|----------------|------|-----|------|
| **Preț** | 0€/lună | 5€/lună | 10€/lună |
| Proprietăți incluse | 1 | 2 | 2 |
| Vehicule incluse | 1 | 2 | 2 |
| Utilizatori | 1 | 10 | 10 |
| Proprietăți suplimentare | — | +2,50€/lună | +5€/lună |
| Vehicule suplimentare | — | +1,50€/lună | +1,50€/lună |
| Înregistrare cheltuieli | ✓ | ✓ | ✓ |
| Remindere automate | ✗ | ✓ | ✓ |
| Bibliotecă documente | ✗ | ✓ | ✓ |
| Invitații rezidenți | ✗ | ✗ | ✓ |

### Restricții plan

::alert{type="info"}
Limitele de plan sunt aplicate la nivel de API — nu pot fi ocolite prin URL direct.
::

- **Plan Free:** maxim 1 proprietate, 1 vehicul, 1 utilizator
- **Plan Pro:** maxim 2 proprietăți + extra plătite, 2 vehicule + extra plătite, 10 utilizatori
- **Plan Pro+:** aceleași limite ca Pro plus acces la funcția **Documente**

### Documente

::alert{type="warning"}
Funcționalitatea Documente este disponibilă exclusiv pentru planul **Pro+**. Accesarea rutei `/dashboard/documents` pe planuri inferioare afișează un ecran de upgrade.
::

::steps

### Selectare plan
Utilizatorul alege un plan din `/dashboard/billing` și apasă „Upgrade".

## 8. Remindere & Notificări

Sistemul de remindere urmărește automat datele importante și trimite notificări prin email înainte de scadentă.

### Domenii de reminder

| Domeniu | Sursa datelor | Offset-uri implicite |
|---------|---------------|---------------------|
| `rca` | Vehicule — câmpul `rcaExpiryDate` | 60, 30, 7, 1, 0 zile |
| `itp` | Vehicule — câmpul `itpExpiryDate` | 60, 30, 7, 1, 0 zile |
| `ro_vignette` | Vehicule — câmpul `vignetteExpiryDate` | 30, 7, 1, 0 zile |
| `bill_due` | Facturi — câmpul `dueDate` | 14, 7, 1, 0 zile |
| `installment_due` | Rate — data scadentă | 14, 7, 1, 0 zile |
| `maintenance` | Mentenanță vehicule — data/km scadent | Configurat per element |

### Setări remindere

::field-group

::field{name="domain" type="enum" required}
Domeniul de reminder (ex: `rca`, `bill_due`).
::

::field{name="enabled" type="boolean"}
Activează sau dezactivează toate reminderele din acest domeniu.
::

::field{name="offsetDays" type="integer[]"}
Lista zilelor de avertizare înainte de scadentă. Modificarea listei anulează ocurențele vechi și creează altele noi.
::

::field{name="timezone" type="string"}
Fusul orar pentru calculul orelor de trimitere. Default: `Europe/Bucharest`.
::

::

### Ciclul de viață al unui reminder

::steps

### Materializare
La fiecare ciclu cron, reminder-service calculează ocurențele viitoare (30, 7, 1 zile înainte de scadentă) și le creează în baza de date cu status `pending`.

### Dispatch
Cron-ul de dispatch (la fiecare ~10 secunde) preia ocurențele `pending` cu `fireAt` ≤ momentul curent și le publică în coada AMQP.

### Trimitere email
notify-service consumă evenimentul și trimite email-ul în limba tenantului (RO sau EN).

### Finalizare
Ocurența este marcată ca `sent`. În caz de eroare, se marchează `failed` (până la 3 reîncercări) sau `dead` (după 3 eșecuri).

::

::alert{type="warning"}
Reminderele sunt trimise asincron. Întârzierea normală de la declanșare la primire este de **sub 1 minut** în mediul de producție.
::

## 9. Dashboard principal

Pagina de start (`/dashboard`) oferă o vedere centralizată a stării gospodăriei.

### Widget-uri KPI

| Widget | Descriere |
|--------|-----------|
| Proprietăți | Numărul total de proprietăți active din tenant |
| Vehicule | Numărul total de vehicule active din tenant |
| Buget curent | Rezumatul bugetului lunii curente (cheltuieli vs. planificat) |
| Sold net | Soldul agregat al tuturor bugetelor active, convertit la moneda de bază |

### Elemente de atenție

Secțiunea „Atenție" surfacează proactiv probleme care necesită acțiune:

| Tip | Condiție de afișare |
|-----|-------------------|
| Buget la risc | Cheltuielile reale depășesc 80% din suma planificată |
| Utilități lipsă | Proprietate activă fără nicio utilitate configurată |
| Conformitate scadentă | Vehicul cu RCA / ITP / rovinietă expirând în fereastra de avertizare |
| Factură restantă | Factură cu data scadentă trecută și neplătită |
| Mentenanță scadentă | Element de mentenanță cu data sau kilometrajul depășit |

### Feed activitate recentă

Listează ultimele acțiuni efectuate în tenant, ordonate cronologic invers, cu:
- Titlul acțiunii (ex: „Proprietate nouă adăugată")
- Entitatea afectată (ex: „Apartament Floreasca")
- Timestamp relativ (ex: „acum 5 minute")

### Conversie valutară

Totalurile din dashboard sunt calculate în moneda de bază a tenantului. Sumele în alte valute sunt convertite automat folosind cursul BNR de la data curentă sau cea mai recentă disponibilă.
